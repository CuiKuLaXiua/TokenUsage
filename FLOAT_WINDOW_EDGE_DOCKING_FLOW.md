# 靠边吸附交互优化说明

## 问题修复

### 修复前的问题
1. 窗口从吸附状态弹出后，拖拽时会立即重新吸附到边缘
2. 无法通过拖拽将窗口从边缘移开

### 修复后的逻辑
现在采用**状态记忆 + 距离清除**的策略：

1. **状态记忆**：窗口首次吸附后，会保存在 `edgeDockState` 中
2. **拖拽时不重新检测**：如果窗口已在 `edgeDockState` 中，拖拽结束时不进行新的边缘检测
3. **距离清除**：拖拽移动超过 200px 时，自动清除 `edgeDockState`，允许重新吸附

---

## 完整交互流程

### 场景 1：首次吸附到边缘

```
用户操作：拖拽窗口到屏幕左侧边缘
  ↓
系统行为：
  1. stop-window-drag 检测到窗口不在 edgeDockState 中
  2. 调用 checkEdgeDocking() 检测到距离左侧 < 20px
  3. 保存 edgeDockState {isDocked: true, edge: 'left', ...}
  4. 动画移动到靠边位置
  5. 启动鼠标轮询
  ↓
结果：窗口靠边隐藏，等待鼠标移入
```

### 场景 2：鼠标移入弹出

```
用户操作：鼠标移动到左侧边缘（距离窗口 5px 内）
  ↓
系统行为（轮询检测）：
  1. 检测到 state.isDocked === true
  2. 检测到鼠标在边缘附近
  3. 动画弹出到原始位置
  4. 更新 state.isDocked = false
  ↓
结果：窗口弹出显示
```

### 场景 3：拖拽弹出的窗口（关键修复）

```
用户操作：拖拽弹出的窗口向右移动（移动距离 < 200px）
  ↓
系统行为：
  1. window-drag-move 实时移动窗口
  2. 拖拽距离 < 200px，不清除 edgeDockState
  3. stop-window-drag 检测到窗口已在 edgeDockState 中
  4. 不进行新的边缘检测
  ↓
结果：窗口自由移动，不会立即重新吸附
```

### 场景 4：拖拽窗口远离边缘

```
用户操作：拖拽窗口向右移动（移动距离 > 200px）
  ↓
系统行为：
  1. window-drag-move 实时移动窗口
  2. 拖拽距离 > 200px，清除 edgeDockState
  3. 停止鼠标轮询
  4. stop-window-drag 检测到窗口不在 edgeDockState 中
  5. 如果窗口不在边缘附近（> 20px），不进行吸附
  ↓
结果：窗口完全自由，不再属于任何边缘
```

### 场景 5：鼠标移出后自动收起

```
用户操作：鼠标从弹出的窗口移开（距离 > 50px）
  ↓
系统行为（轮询检测）：
  1. 检测到 state.isDocked === false（但仍在 edgeDockState 中）
  2. 检测到鼠标远离窗口（> 50px）
  3. 动画收起到靠边位置
  4. 更新 state.isDocked = true
  ↓
结果：窗口自动收起
```

### 场景 6：重新吸附到另一侧边缘

```
用户操作：
  1. 拖拽窗口从左侧边缘向右移动 250px（清除左侧 edgeDockState）
  2. 拖拽窗口到屏幕右侧边缘
  ↓
系统行为：
  1. stop-window-drag 检测到窗口不在 edgeDockState 中
  2. 调用 checkEdgeDocking() 检测到距离右侧 < 20px
  3. 保存新的 edgeDockState {isDocked: true, edge: 'right', ...}
  4. 动画移动到右侧靠边位置
  5. 启动鼠标轮询
  ↓
结果：窗口吸附到右侧边缘
```

## 关键阈值说明

| 参数 | 值 | 说明 |
|------|-----|------|
| `EDGE_THRESHOLD` | 20px | 距离边缘多少像素内触发吸附 |
| `EDGE_REVEAL_ZONE` | 5px | 鼠标距离窗口多少像素内触发弹出 |
| `EDGE_HIDE_ZONE` | 50px | 鼠标距离窗口多少像素外触发收起 |
| `CLEAR_DOCK_THRESHOLD` | 200px | 拖拽移动多少像素后清除吸附状态 |

## 状态管理

### EdgeDockState 结构
```typescript
interface EdgeDockState {
  isDocked: boolean      // true: 靠边隐藏中, false: 已弹出但仍在边缘附近
  edge: 'left' | 'right' | 'top' | null  // 吸附的边缘方向
  dockX: number          // 靠边隐藏时的 X 坐标
  dockY: number          // 靠边隐藏时的 Y 坐标
  originalX: number      // 弹出后的 X 坐标（原始位置）
  originalY: number      // 弹出后的 Y 坐标（原始位置）
}
```

### 状态转换图
```
[无状态] 
  ↓ 首次吸附
[isDocked: true]
  ↓ 鼠标移入
[isDocked: false]
  ↓ 鼠标移出 → [isDocked: true] (收起)
  ↓ 拖拽 < 200px → [isDocked: false] (保持自由)
  ↓ 拖拽 > 200px → [无状态] (清除)
  ↓ 释放到新边缘 → [isDocked: true] (重新吸附)
```

---

## 优势

1. **更自然的交互**
   - 用户拖拽弹出的窗口时，不会立即被"吸"回边缘
   - 需要明确的意图（拖拽到新边缘）才会重新吸附

2. **状态清晰**
   - 拖拽距离明确区分"在边缘附近移动"和"离开边缘"
   - 200px 阈值提供足够的操作空间

3. **灵活性**
   - 可以轻松地将窗口从一个边缘移到另一个边缘
   - 可以完全取消吸附状态（拖到屏幕中间）

4. 释放鼠标

8. ✅ 验证：窗口吸附到顶部边缘
